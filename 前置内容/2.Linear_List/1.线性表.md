### 数据结构基础知识补充
数据结构是相互之间存在一种或多种特定关系的数据元素的集合，这里的特定关系即为结构，而数据之间常常有四种基本结构：集合、线性结构、树状结构以及图状结构。这里的线性表就属于线性结构

什么是线性结构:
1. 存在唯一的一个被称为"第一个"的数据元素
2. 存在唯一的一个被称为"最后一个"的数据元素
3. 除第一个之外，集合中的每个数据元素均只有一个前驱
4. 除了最后一个之外，集合中的每个元素均只有一个后继

### 什么是线性表 linear List

线性表是具有相同数据类型的 n (n > 0) 个数据元素的有限序列，其中 n 为表长，当 n = 0 时，线性表是一个空表。弱用L命名线性表一边表示为：

L = （a1,a2,a3,...,an）

线性表的基本操作
1. InitList(&L): 初始化表。构造一个空的线性表L，分配内存空间
2. DestroyList(&L): 销毁操作。销毁线性表，并释放线性表L所占用的内存空间
3. ListInsert(&L,i,e): 插入操作。在L中的第i个位置插入指定元素e
4. ListDelete(&L,i,&e): 删除操作。删除表L中第i个位置的元素，并用e返回删除元素的值
5. LocateElem(L,e): 按值查找
6. GetElem(L,i): 按位查找
7. Length(L): 求表长
8. PrintList(L): 输出操作。按前后顺序输出线性表L的所有元素值
9. Empty(L): 判空操作。若L为空表，则返回true，否则返回false


其实这里面的核心就是增删改查，其他的是方便我们对这个数据结构进行调用等等功能的
可以根据我们实际的工作需求定义其他的基本操作

如果需要带回参数修改的话  就要用引用类型(也就是地址符)

##### 为什么要定义数据结构的基本操作
1. 团队合作，定义的数据结构要让别人能够方便使用。
2. 将常用操作/计算封装成函数，避免重复工作，降低出错风险。

###### 顺序表
用顺序存储的方式实现线性表。顺序存储就是吧逻辑上相邻的元素存储在物理位置上也相邻的存储单元，元素间的关系由存储单元的邻接关系来体现

线性表的顺序表示：用一组地址连续的存储单元依次存储线性表的数据元素

设每个数据元素占用x个存储单元，线性表第i个元素存储在地址为 LOC(ai)的位置，则第i+1个元素存储在LOC(ai) + x 的位置，LOC(a1) 是线性表的第一个数据元素a1的存储位置，通常称为线性表的起始位置或者基地址。
